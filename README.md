# 个人简历网站（单页 · 可编辑）

> **给访客 / 站长（怎么用网页）**：请阅读 **[使用手册.md](./使用手册.md)** —— 介绍页面布局、主题与站点编辑、如何插入图片与添加作品等。  
> 本文 `README.md` 侧重开发者：安装、运行、环境变量与部署。

基于 **Next.js 15** 与 **React 19** 的单页个人简历 / 作品集站点：纵向滚动浏览「首屏 → 履历 → 作品」三个区块，支持多套主题、本机持久化内容，并在授权网络下提供可视化编辑能力。

## 适合谁用

- 需要一份**可在线展示**的个人介绍、履历时间线与作品栅格。
- 希望**在浏览器里直接改文案与条目**，而不必改代码再部署（数据写在当前浏览器的本地存储中）。
- 愿意自行部署到 Vercel 等平台，或本地运行后用内网穿透临时分享。

## 安装与运行

### 1. 安装依赖

在仓库根目录（含 `package.json` 的目录）执行：

```bash
npm install
```

### 2. 本地开发

```bash
npm run dev
```

浏览器打开 **http://localhost:3000** 即可访问。

若希望同一局域网内的手机访问本机服务，可使用：

```bash
npm run dev:lan
```

然后用终端里提示的 **Network** 地址（例如 `http://192.168.x.x:3000`）在手机浏览器打开（需防火墙放行 3000 端口）。

### 3. 生产构建与启动

```bash
npm run build
npm run start
```

默认监听 **http://localhost:3000**。部署到云平台时，请按平台文档配置 `ALLOWED_EDIT_IPS` 等环境变量（见下文）。

**GitHub → 阿里云轻量自动部署**：推送 `main`/`master` 后由 GitHub Actions 自动构建并重启，配置步骤见 **[docs/DEPLOY-AUTO.md](docs/DEPLOY-AUTO.md)**。

## 浏览与导航

| 功能 | 说明 |
|------|------|
| **分区** | 单页纵向三块：**首屏**（姓名、一句话、形象照）、**履历**（经历与教育卡片）、**作品集**（封面栅格与外链）。 |
| **顶栏锚点** | 页面顶部固定导航：**首屏 / 履历 / 作品**，点击平滑滚动到对应区块（`html` 已配置平滑滚动）。 |

| **作品卡片** | 点击主图区域会在**新标签页**打开你配置的作品链接。 |

## 如何编辑内容

编辑入口是否显示，由服务端接口 **`/api/can-edit`** 根据环境变量与访问 IP 决定（见下一节）。**有编辑权限时**，你会看到：

### 右下角「站点编辑」

固定于**右下角**（与左下角「主题」对称），点击展开菜单：

| 菜单项 | 作用 |
|--------|------|
| **首屏与形象** | 打开资料弹窗：姓名、一句话介绍、形象照（URL 或上传小图）、首屏辅助文案（眉标、滚动提示、照片说明等）。 |
| **履历页用词** | 打开文案弹窗：履历页眉、主标题、简介、各分区标题、卡片底部引导语、详情页用语等。 |
| **作品页用词** | 打开文案弹窗：作品页眉、主标题、说明段落，以及卡片上的「打开链接」、海报区说明等文字。 |

弹窗内修改后点 **保存**，会写入**当前浏览器**的站点快照（与 `localStorage` 机制一致）。

### 履历页内联操作

- **经历卡片**：每条经历右侧有 **「编辑」**，打开经历编辑器（职位、公司、时间、成果、代表项目等）。
- **教育背景**：分区标题旁有 **「编辑 / 保存」**，展开教育条目编辑面板。

### 作品集

- **「+ 插入作品」**：按表单填写标题、简介、封面图 URL、可选海报 URL、外链后添加。
- **「删除」**：在可编辑状态下，卡片右上角可删除该条作品。

### 首次进入（有编辑权限时）

若从未完成过引导，可能会自动弹出 **资料设置**；也可随时通过「站点编辑 → 首屏与形象」再次打开。

## 数据保存在哪里

- **访客看到的正式版**：服务器项目根目录 **`data/published-site.json`**（通过 `GET/PUT /api/site`）。站长在白名单 IP 下编辑并保存时会自动发布到此文件。
- **站长本机草稿**：浏览器 **`localStorage`** 键名 **`resume-site-bundle-v2`**（见 `lib/persist-site.ts`），含 `savedAt` 时间戳；若本机比服务器新，刷新后会优先恢复本机并自动重试发布。
- **备份建议**：定期备份 `data/published-site.json`；亦可导出本机 localStorage 中的 JSON。

## 谁能编辑（IP 白名单）

接口：`GET /api/can-edit`  
返回 JSON 字段示例：`{ "canEdit": true, "ip": "…", "reason": "…" }`。

项目根目录提供 **`.env.example`**，可复制为 **`.env.local`** 后按需填写：

| 变量 | 含义 |
|------|------|
| **`ALLOWED_EDIT_IPS`** | 逗号分隔的 **公网 IPv4/IPv6**。仅列表中的访问者会看到编辑相关 UI 并允许保存；未命中则为只读（顶栏会提示「只读浏览」类说明）。 |
| **`ALLOW_EDIT_IN_DEV`** | 设为 `true` 且 **`NODE_ENV=development`** 时：仅 **环回地址**（本机）可编辑；可与白名单组合使用。 |

补充行为（实现见 `app/api/can-edit/route.ts`）：

- **开发环境**且 **未配置** `ALLOWED_EDIT_IPS`：默认 **允许所有人编辑**（便于本地调试）；**生产环境请务必配置白名单**。
- 部署在 **Nginx / CDN / Vercel** 等反向代理后，需保证请求头携带 **`x-forwarded-for`**（或 `x-real-ip`），否则可能无法解析真实客户端 IP。

查看当前请求在服务端解析到的出口 IP（便于填白名单）：访问 **`/api/my-ip`**（若项目中有该路由）。

## 分享给朋友预览

- **同一 WiFi**：使用 `npm run dev:lan`，把手机/朋友的电脑连到同一网络，访问终端显示的 **Network** 地址即可。
- **互联网临时链接**：在本机已运行 `npm run dev` 的前提下，可另开终端执行：

  ```bash
  npx --yes cloudflared tunnel --url http://localhost:3000
  ```

  终端会打印 **`https://xxxx.trycloudflare.com`** 形式的地址，发给对方即可。**关闭该命令后链接即失效**；每次运行子域名通常不同。

  若浏览器出现 **`530 The origin has been unregistered from Argo Tunnel`**：这是 **Cloudflare 隧道源站已注销**（`cloudflared` 进程已退出、断线，或临时隧道会话结束），**不是项目代码错误**。处理步骤：① 确认本机 **`npm run dev` 仍在运行**，且端口与隧道一致（例如均为 `3000`）；② **重新执行**上面的 `cloudflared` 命令，使用终端里**新出现**的 `https://xxxx.trycloudflare.com` 访问，旧链接不可再用。

  若出现 **`1033 Cloudflare Tunnel error`**：表示边缘**无法连上**你本机正在运行的 `cloudflared`（或隧道名已失效）。常见原因：`cloudflared` 未启动/已崩溃、仍使用**上一次运行**的旧 trycloudflare 地址、本机防火墙/代理拦截出站、或 `npm run dev` 未在 `--url` 所指端口监听。**不是 Next 项目代码问题**。处理：先启动 `npm run dev`，再**重新**执行 `cloudflared` 命令，**只用本次终端新打印的 URL**；不要用收藏夹里的旧链接。

- **正式对外发布**：建议将项目部署到 **Vercel** 等平台，绑定自己的域名，并在后台配置 `ALLOWED_EDIT_IPS`，避免陌生人随意改你的线上内容。

## 常见问题

**Q：朋友打不开我发的 trycloudflare 链接？**  
A：确认本机 `npm run dev` 与 `cloudflared` 两个进程都在运行；让对方换网络或过几秒重试。

**Q：页面显示 530 / The origin has been unregistered from Argo Tunnel？**  
A：隧道已从 Cloudflare 侧注销，见上文「分享给朋友预览」中对该错误的说明；重启 `cloudflared` 并改用**新 URL**。

**Q：页面显示 1033 / Cloudflare Tunnel error？**  
A：边缘连不上本机 `cloudflared` 或隧道已失效，见上文同节对 **1033** 的说明；勿使用过期 trycloudflare 链接。

**Q：朋友能看到编辑按钮，我不想让他们改？**  
A：在生产环境配置 **`ALLOWED_EDIT_IPS`** 仅包含你自己的公网 IP；开发环境不要长期用「未配白名单默认可编辑」的方式对外演示。

**Q：换电脑后内容没了？**  
A：数据在**原浏览器**的 `localStorage`，不会随仓库同步。需要在新环境重新填写，或自行迁移备份的 JSON。

**Q：作品图片外链不显示？**  
A：检查图片 URL 是否允许跨站引用、是否 HTTPS；Next 图片组件对外链域名可能需在 `next.config.ts` 中配置 `images.remotePatterns`（若你扩展了域名）。
