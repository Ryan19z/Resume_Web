## 2026-06-16 1) PDF 解析结果严重缺失（工作经历为 0、项目无要点）

- **现象**  
  导入后出现：工作经历为空、项目仅一句话、详情内容空白。
- **根因**  
  规则解析对日期行、项目续行、要点合并和章节识别覆盖不足；部分 PDF 文本结构被误分段。
- **修复**  
  增强规则引擎：  
  - 支持 `YYYY年M月-YYYY年M月` 等日期格式  
  - 增强公司行/职位行识别  
  - 合并项目 bullet 续行  
  - 兼容无日期项目标题、逗号分隔技能、个人总结分区  
  - 去重项目时保留内容更丰富版本
- **验证**  
  多份样本脚本回归通过（含侯玉婷样本），导入后工作/项目恢复可读结构。

## 2026-06-16 2) 导入新简历后“核心亮点”残留上一人数据

- **现象**  
  更换简历导入后，首页核心亮点仍显示上一位候选人内容。
- **根因**  
  导入 patch 未全量覆盖部分字段；空数组在持久化层被回退到默认值；Hero 自动保存存在竞态。
- **修复**  
  - 导入时强制清空并覆盖：核心亮点、证件照、右侧展示、联系二维码、重点展示媒体  
  - 持久化层允许空数组落盘，不再回退默认  
  - Hero 自动保存增加跳过首帧逻辑，规避导入瞬间覆盖
- **验证**  
  连续导入不同简历，旧亮点不再残留。

## 2026-06-16 3) AI 解析方式不透明（用户不知道本次是否用 AI）

- **现象**  
  仅显示“已配置 AI”，无法判断本次实际是 AI 还是规则引擎。
- **根因**  
  前端只展示能力状态，缺少“本次解析方式”和“回退状态”。
- **修复**  
  - 预览/成功页增加解析方式徽章：AI / 规则 / AI失败回退  
  - API 返回 `llmProvider`、`llmModel`、`llmFallback`  
  - 回退时加明显警告文案
- **验证**  
  不同场景（AI成功/未配置/回退）均能正确显示状态。

## 2026-06-16 4) 置信度几乎总是 92%

- **现象**  
  上传不同简历，置信度长期固定 92%。
- **根因**  
  LLM 路径置信度写死为 `0.92`；规则路径也有 0.92 封顶。
- **修复**  
  - 新增 `computeParseConfidence(parsed)` 按字段完整度动态评分  
  - 融合内容充实度（经历要点、项目要点、教育完整性）  
  - warnings 触发轻度扣分  
  - UI 文案从“置信度”调整为“字段完整度”
- **验证**  
  完整/稀疏样本分数显著分离，不再固定 92%。

## 2026-06-16 5) 项目经历只有一句话（详情信息丢失）

- **现象**  
  项目卡片和详情仅一段描述，缺少多条职责/成果要点。
- **根因**  
  LLM 的 project schema 未强制 `bullets`；清洗层未完整吸收项目要点字段。
- **修复**  
  - 扩充 LLM `PARSE_SCHEMA`：`role` / `period` / `bullets`  
  - 提示词明确要求“项目要点拆条”  
  - `sanitizeParsed` 接收并标准化 bullets（支持字符串拆分）
- **验证**  
  项目详情可展示多条 keyResults，不再单句化。

## 2026-06-16 6) 校园经历/实践经历误入工作经历，分类不稳定

- **现象**  
  学生会/班委/竞赛角色常被放进工作经历；“实践经历”章节有时丢失或不分类。
- **根因**  
  模型输出语义边界不清；非标准字段（practice/activities）未统一收敛。
- **修复**  
  - 强化 prompt 与 schema 的三类边界：  
    - `experience`：实习/正式工作  
    - `education.campusExperiences`：在校职务与校园实践  
    - `projects`：具体项目/课题/竞赛作品  
  - 增加实践经历别名兜底归类
  - 新增后端 `autoClassifyResumeContent` 自动分流策略
- **验证**  
  “实践经历”不再整体丢失，能进入三类之一。

## 2026-06-16 7) 工作经历与项目经历重复展示同一条内容

- **现象**  
  同一实践内容同时出现在工作经历和项目经历。
- **根因**  
  自动分类策略早期包含“工作优先 + 项目镜像”逻辑。
- **修复**  
  移除镜像策略，改为“一条经历只归一个主类”，禁止双写。
- **验证**  
  重新导入后不再出现跨区重复条目。

## 2026-06-16 8) 项目经历区块没有添加/编辑/删除

- **现象**  
  工作经历有编辑按钮，项目经历没有增删改能力。
- **根因**  
  页面对 `projectExperience` 区块禁用了编辑入口；上下文缺少对应 CRUD API。
- **修复**  
  - `SiteContentProvider` 增加：  
    - `addProjectExperienceItem`  
    - `removeProjectExperienceItem`  
    - `updateProjectExperienceItem`  
  - `ResumePage` 接入项目经历增删改按钮  
  - `ExperienceEditorModal` 支持 `section=projectExperience`
- **验证**  
  项目经历可完整新增、编辑、删除。

## 2026-06-16 9) 预览模式展示空模块（空作品集仍显示）

- **现象**  
  预览给 HR 时，即使无作品也显示“作品集”空区块。
- **根因**  
  页面结构固定渲染；导航锚点不感知数据为空。
- **修复**  
  - 预览模式下按内容条件渲染 Resume/Portfolio  
  - 顶部锚点导航同步隐藏无内容模块入口  
  - 履历页内部空子区块（工作/项目/教育）也按条件隐藏
- **验证**  
  预览页面仅显示有内容模块。

## 2026-06-16 11) 项目卡片外层看似“无内容”，详情里却有内容

- **现象**  
  列表卡片正文为空，但点进详情能看到多条成果。

- **验证**  
  外层列表与详情内容一致，不再“空卡片”。

## 2026-06-16 14) “近期经历”缺少最新项（施工经理未出现）

## 2026-06-16 15) 导入后经历排序不符合阅读习惯

- **现象**  
  期望“最早在上，最新在下”，但导入顺序不统一。
- **根因**  
  映射层未统一按 period 起始时间排序。
- **修复**  
  在 `resume-parse-mapper.ts` 增加通用排序函数，统一应用到：  
  - 工作经历  
  - 项目经历  
  - 教育背景
- **验证**  
  重新导入后三类列表顺序一致（时间正序）。
